# Contributing to AxialNet

Thanks for taking the time to contribute! Every bit helps — whether it's a bug fix, a new feature, or improving docs.

---

## Getting started

1. **Fork** the repository
2. **Clone** your fork locally
   ```bash
   git clone https://github.com/your-username/repo-name.git
   ```
3. **Create a branch** for your change
   ```bash
   git checkout -b fix/your-fix-name
   ```
4. **Make your changes**, then commit with a clear message
   ```bash
   git commit -m "fix: describe what you changed"
   ```
5. **Push** and open a Pull Request against `main`

---

## Guidelines

- Keep PRs focused — one change per PR
- Write clear commit messages (we follow [Conventional Commits](https://www.conventionalcommits.org))
- Add or update tests if your change affects logic
- Update docs if your change affects behaviour

---

## Reporting bugs

Open an [issue](https://github.com/axialnet/.github/issues) and include:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Your environment (OS, version, etc.)

---

## Suggesting features

Open an issue with the `feature request` template. Describe the problem you're trying to solve, not just the solution — it helps us discuss the best approach.
